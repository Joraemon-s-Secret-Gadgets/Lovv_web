# 1차 MVP 목업 User Flow / Service Flow

## Scope

| 구분 | 내용 |
| --- | --- |
| 목적 | Lovv 1차 MVP 목업에서 사용자가 온보딩부터 AI 일정 생성 결과까지 이동하는 흐름을 정리한다. |
| 기준 화면 | 진입 전 온보딩, 메인 화면, AI 일정 챗봇, 챗봇 아래 생성 일정 상세 영역 |
| 핵심 방향 | 첫 진입 전 온보딩으로 취향을 받고, 메인 진입 후 `AI 일정 짜기`에서 챗봇 기반 여행 생성 작업을 진행한다. |
| MVP 처리 | 백엔드, 실제 AI API, 지도 API 없이 React 상태와 `localStorage` 기반 목업으로 동작한다. |
| 제외 범위 | 로그인, 계정 저장, 실제 지도/장소 검색, 실제 AI 응답 생성, 전체 채팅 로그 저장 |

## User Flow

| 단계 | 화면 / 상태 | 사용자 행동 | 시스템 반응 | 사용자에게 보이는 결과 | 완료 조건 |
| --- | --- | --- | --- | --- | --- |
| 1 | 최초 진입 | 사용자가 Lovv 웹앱에 접속한다. | `lovv.preference` 저장 여부를 확인한다. | 저장된 취향이 없으면 메인 전에 온보딩 화면이 먼저 열린다. | 최초 사용자는 메인 화면을 보기 전에 온보딩을 만난다. |
| 2 | 진입 전 온보딩 | 사용자가 여행 분위기 카드 중 하나를 선택한다. | 선택된 취향을 화면 상태에 반영한다. | 선택 카드가 강조되고, 하단에 취향 시그널 칩이 표시된다. | 사용자가 선택한 카드가 명확히 구분된다. |
| 3 | 진입 전 온보딩 | 사용자가 `이 취향으로 Lovv 시작하기`를 누른다. | 선택한 `cityPair`만 `localStorage`에 저장한다. | 메인 화면으로 전환된다. | 취향 저장 후 메인 화면에 진입한다. |
| 4 | 메인 화면 | 사용자가 개인화된 메인 화면을 확인한다. | 저장된 취향을 기반으로 메인 문구와 추천 맥락을 구성한다. | Lovv 캐릭터, 서비스 카드, `AI 일정 짜기` 버튼이 보인다. | 사용자는 현재 선택된 여행 취향을 인지한다. |
| 5 | 메인 화면 | 사용자가 `AI 일정 짜기`를 클릭한다. | 온보딩을 다시 띄우지 않고 챗봇 워크스페이스로 이동한다. | `AI 일정 챗봇` 화면이 열린다. | AI 일정 생성 플로우가 시작된다. |
| 6 | 챗봇 워크스페이스 | 사용자가 첫 안내 메시지를 읽는다. | 저장된 취향을 챗봇 초기 메시지와 상태 칩에 반영한다. | 선택 취향 기반의 질문, 추천 방향, 진행 상태가 보인다. | 사용자는 어떤 정보를 입력해야 하는지 이해한다. |
| 7 | 챗봇 워크스페이스 | 사용자가 `당일치기`부터 `4박 5일`까지 기간 가이드 칩 중 하나를 먼저 고른다. | 선택한 기간을 대화 목록에 추가하고 일정 초안 기간에 반영한다. | 사용자 말풍선과 하단 일정 제목이 선택 기간에 맞게 바뀐다. | 사용자는 가능한 일정 범위를 추측하지 않고 선택한다. |
| 8 | 챗봇 워크스페이스 | 사용자가 메시지를 전송한다. | MVP용 deterministic 응답과 일정 초안을 생성한다. | AI 응답 말풍선과 간단한 일정 초안 카드가 표시된다. | 챗봇 안에서 여행 생성이 진행되는 느낌을 준다. |
| 9 | 챗봇 아래 일정 상세 | 사용자가 챗봇 아래로 이어지는 결과 영역을 확인한다. | 생성된 초안을 기반으로 시간대별 상세 일정을 렌더링한다. | 오전, 오후, 저녁 일정 블록과 이동 힌트, 추천 이유가 보인다. | 사용자는 대화 결과를 결과 페이지처럼 확인한다. |
| 10 | 챗봇 워크스페이스 | 사용자가 추가 요청을 입력한다. | 대화 메시지를 추가하고, 일정 초안 또는 상세 내용을 갱신한다. | 같은 화면에서 대화와 일정 상세가 이어진다. | 사용자는 화면 이동 없이 일정을 조정할 수 있다. |
| 11 | 재방문 | 사용자가 웹앱에 다시 접속한다. | 저장된 `lovv.preference`를 읽는다. | 온보딩을 건너뛰고 메인 화면으로 진입한다. | 재방문자는 바로 서비스 메인으로 들어간다. |
| 12 | 저장값 오류 | 저장된 취향 값이 없거나 깨져 있다. | 잘못된 저장값을 무시한다. | 온보딩 화면으로 돌아간다. | 사용자는 안전하게 다시 취향을 선택한다. |

## Service Flow

| 단계 | Trigger | Frontend 처리 | 상태 / 데이터 | 목업 로직 | 결과 |
| --- | --- | --- | --- | --- | --- |
| 1 | 앱 로드 | `localStorage.getItem('lovv.preference')`를 호출한다. | `lovv.preference` | 저장된 `cityPair`가 기존 취향 목록에 있는지 검증한다. | 유효하면 `home`, 없거나 오류면 `onboarding`으로 시작한다. |
| 2 | 온보딩 카드 선택 | 선택된 취향을 React state에 저장한다. | `selectedPreference` | 카드 선택 상태와 시그널 칩을 즉시 갱신한다. | 선택 피드백이 화면에 반영된다. |
| 3 | 온보딩 완료 | 선택한 취향의 `cityPair`만 저장한다. | `localStorage: lovv.preference` | 전체 사용자 정보나 채팅 로그는 저장하지 않는다. | 메인 화면 렌더링에 필요한 최소 취향만 남는다. |
| 4 | 메인 렌더링 | 저장된 취향을 메인 화면 UI에 주입한다. | `selectedPreference` | 선택 취향의 설명, 태그, route hint를 표시한다. | 개인화된 메인처럼 보인다. |
| 5 | `AI 일정 짜기` 클릭 | `activeView`를 `chat`으로 변경한다. | `activeView: 'chat'` | 온보딩 모달이나 게이트를 다시 호출하지 않는다. | 챗봇 워크스페이스가 열린다. |
| 6 | 챗봇 초기화 | 초기 assistant 메시지와 진행 상태를 구성한다. | `ChatMessage[]`, `selectedPreference` | 취향별 route hint와 signals를 문장에 포함한다. | 사용자가 입력할 조건을 안내한다. |
| 7 | 메시지 입력 | 입력값을 trim하고 빈 값 여부를 검사한다. | `inputValue` | 빈 문자열 또는 공백만 있는 입력은 전송하지 않는다. | 불필요한 빈 말풍선이 생기지 않는다. |
| 8 | 메시지 전송 | 사용자 메시지를 대화 배열에 append한다. | `ChatMessage[]` | 메시지는 화면 상태에만 저장한다. | 사용자 말풍선이 대화창에 추가된다. |
| 9 | 응답 생성 | deterministic assistant 응답을 append한다. | `ChatMessage[]`, `PlanDraft` | 실제 AI API 대신 선택 취향과 사용자 입력 키워드 기반 고정 응답을 만든다. | 목업에서도 AI가 반응하는 흐름이 보인다. |
| 10 | 일정 초안 생성 | `PlanDraft`를 만든다. | `cityPair`, `mood`, `summary`, `stops[]` | 오전, 오후, 저녁 최소 3개 stop을 생성한다. | 챗봇 내부 또는 하단에 초안 카드가 표시된다. |
| 11 | 일정 상세 렌더링 | 챗봇 패널 아래 `생성된 일정 상세` 영역을 그린다. | `PlanDraft.stops[]` | 시간대, 장소 성격, 이동 힌트, 추천 이유를 timeline 형태로 보여준다. | 결과 페이지처럼 읽히는 상세 일정이 표시된다. |
| 12 | 추가 요청 처리 | 새 사용자 메시지를 기존 흐름에 append한다. | `ChatMessage[]`, optional `PlanDraft` | 기간은 `당일치기`, `1박 2일`, `2박 3일`, `3박 4일`, `4박 5일` 범위에서 인식하고, "덜 걷게", "카페 추가" 같은 키워드에 따라 목업 문구를 조정할 수 있다. | 같은 화면에서 일정 조정 흐름이 이어진다. |
| 13 | 개인정보 보호 | 저장 전 데이터를 제한한다. | `lovv.preference`, optional `lovv.planDraft` | 전체 채팅 transcript는 저장하지 않는다. | MVP에서도 개인정보 저장 범위가 작게 유지된다. |
| 14 | 오류 복구 | JSON parse 실패 또는 알 수 없는 취향 값 발생 | storage read result | 오류를 삼키고 기본 온보딩으로 fallback한다. | 사용자가 깨진 상태에 갇히지 않는다. |

## MVP Mockup Screen Map

| 화면 | 역할 | 주요 UI | 다음 이동 |
| --- | --- | --- | --- |
| 진입 전 온보딩 | 메인 진입 전 취향 기준 설정 | 도시 분위기 카드, 취향 시그널, 시작 버튼 | 메인 화면 |
| 메인 화면 | Lovv 서비스 첫 화면과 기능 진입점 | 캐릭터, 개인화 문구, `AI 일정 짜기`, 보조 기능 버튼 | AI 일정 챗봇 |
| AI 일정 챗봇 | 여행 조건을 대화로 입력하는 작업 공간 | assistant/user 말풍선, 기간 가이드 칩, 입력창, 전송 버튼, 상태 칩 | 생성 일정 상세 |
| 생성 일정 상세 | 챗봇 결과를 결과 페이지처럼 확인하는 영역 | 일정 제목, 시간대별 timeline, 이동 힌트, 추천 이유 | 추가 대화 또는 추후 저장/비교 |

## Data Policy

| 데이터 | 저장 위치 | 저장 여부 | 이유 |
| --- | --- | --- | --- |
| 선택 취향 `cityPair` | `localStorage.lovv.preference` | 저장 | 재방문 시 온보딩을 건너뛰고 개인화 메인을 보여주기 위해 필요하다. |
| 챗봇 전체 대화 | React state | 미저장 | MVP에서 로그인 없이 동작하므로 불필요한 개인정보 저장을 피한다. |
| 생성 일정 초안 | React state, optional `localStorage.lovv.planDraft` | 선택 저장 | 새로고침 후 결과 유지가 필요할 때만 lightweight draft로 제한한다. |
| 실제 위치 / 지도 데이터 | 없음 | 미저장 | 1차 MVP 목업에서는 지도 API를 제외한다. |
| 로그인 정보 | 없음 | 미저장 | 1차 MVP 범위가 아니다. |
