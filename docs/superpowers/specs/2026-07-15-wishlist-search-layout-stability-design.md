# Wishlist Search and Desktop Layout Stability Design

## Summary

노트북·PC의 세부 일정 화면에서 Kakao Places 맛집 검색 후보를 5개에서 10개로 확대하고, 선택한 장소 카드가 우측 sticky 영역의 높이 제약 때문에 잘리지 않도록 위시리스트 목록의 스크롤과 카드 축소 규칙을 명확히 한다.

## Problem

1. `searchKakaoMealPlaces`가 정상 Kakao 결과를 `.slice(0, 5)`로 제한해 선택지가 좁아 보인다.
2. 데스크톱 우측 열은 뷰포트 기반 최대 높이 안에서 지도와 위시리스트를 함께 배치한다.
3. 위시리스트 카드에 이미지·주소·전화·동작 버튼이 포함되면 목록과 카드가 사용 가능한 높이보다 커져 카드 하단이 잘려 보일 수 있다.

## Goals

- Kakao Places의 한 번 검색에서 최대 10개 후보를 보여준다.
- 검색 결과는 기존 검색 팝업 내부에서 스크롤한다.
- 데스크톱 위시리스트는 우측 패널 안에서 자체 스크롤한다.
- 선택한 장소 카드 한 개의 테두리, 내용, 위치 선택·제거 버튼이 잘리지 않는다.
- 노트북 높이에서도 지도와 위시리스트가 같은 우측 열에 유지된다.
- 기존 드래그·클릭 위치 지정과 장소 제거 동작을 유지한다.

## Non-goals

- Kakao Places 다음 페이지 호출 또는 무한 스크롤을 추가하지 않는다.
- 검색 팝업을 drawer나 전체 화면 modal로 재설계하지 않는다.
- 백엔드 API와 Kakao 이미지 API 계약을 변경하지 않는다.
- 모바일 레이아웃을 재설계하지 않는다.

## Design

### Search results

- `kakaoMealSearch.ts`의 정상 결과 제한을 5개에서 10개로 변경한다.
- 변환 및 유효성 필터링 후 첫 10개를 사용한다.
- 동일 검색어 캐시와 오류 상태는 유지한다.
- 검색 후보 목록의 기존 `max-height`와 내부 스크롤은 유지한다.

### Desktop right column

- 우측 지도·위시리스트 grid에서 두 번째 행이 부모의 사용 가능한 높이를 초과하지 않도록 최소 크기와 overflow 경계를 명시한다.
- 위시리스트 컨테이너는 넘치는 목록을 직접 잘라내지 않고 내부 목록에 스크롤을 위임한다.
- 위시리스트 목록은 `flex: 1`, `min-height: 0`, `overflow-y: auto` 조합으로 남은 공간을 사용한다.
- 데스크톱에서는 목록 최대 높이를 노트북 뷰포트에 맞는 범위로 제한하고, 태블릿 이하의 기존 최대 높이는 유지한다.

### Wishlist cards

- 각 장소 카드는 스크롤 목록 안에서 축소되지 않도록 한다.
- 이미지, 정보, 동작 버튼의 데스크톱 배치는 기존 flex 흐름을 유지하되 카드 축소를 막아 안정화한다.
- 장소명과 주소는 긴 텍스트가 가로 폭을 밀어내지 않도록 `min-width: 0`과 줄바꿈을 유지한다.
- 주소는 최대 2줄로 제한하되 장소명, 위치 선택, 제거 버튼은 항상 표시한다.

## Error Handling

- Kakao 키 누락, SDK 오류, 검색 결과 없음 안내는 기존 동작을 유지한다.
- 10개 중 일부 결과가 유효하지 않으면 유효한 결과만 표시한다.
- 이미지 조회 실패 시 이미지 없이 카드 정보를 표시한다.

## Accessibility

- 검색 결과와 담아둔 맛집 목록의 기존 접근 가능한 이름을 유지한다.
- 내부 스크롤이 생겨도 키보드로 각 링크와 버튼에 접근할 수 있어야 한다.
- 동작 버튼의 이름과 선택 상태(`aria-pressed`)를 유지한다.

## Testing

- Kakao 검색 어댑터가 10개를 반환하고 11번째 결과를 제외하는 단위 테스트를 추가한다.
- 10개 검색 결과가 팝업에 렌더링되는 컴포넌트 테스트를 추가하거나 기존 fixture를 확장한다.
- 선택한 위시리스트 카드에 이미지·긴 주소·전화번호가 있어도 필수 내용과 버튼이 렌더링되는 회귀 테스트를 추가한다.
- 관련 테스트, 전체 프론트 테스트, lint, build를 실행한다.
- 노트북 크기의 실제 Chromium에서 목록 스크롤과 카드 경계를 확인한다.

## Acceptance Criteria

- 검색 결과가 최대 10개 표시된다.
- 11번째 이후 후보는 현재 검색 결과에 표시되지 않는다.
- 데스크톱 우측 패널에서 선택한 장소 카드가 잘리지 않는다.
- 여러 장소가 있으면 위시리스트 목록만 스크롤된다.
- 장소 위치 선택, 제거, 드래그, 코스 사이 클릭 추가가 기존대로 동작한다.
- 관련 자동 테스트와 실제 브라우저 검증이 통과한다.
